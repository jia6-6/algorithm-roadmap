## 🧠 设计决策与避坑注释（必看！）

如果你在阅读路线时觉得某些地方“不合理”或“权重偏颇”，大概率是因为以下设计考量：

### 1. 为什么链表模块权重偏大？
**（答）** 本路线不仅服务竞赛，也兼顾大学生数据结构课业。指针实现链表是期末笔试/机考的高频考点，此处多花时间，是为了让你在竞赛和学业之间少一次“临时抱佛脚”。

### 2. 为什么二叉树阶段推荐力扣（TreeNode*）而非洛谷（数组模拟）？
**（答）** 数组模拟会掩盖递归的返回值传递过程。力扣的指针版强行逼你思考“这个递归到底要返回什么”，是练递归内功的最佳道场。等递归通了，二轮再看数组模拟，一天就会。

### 3. 为什么动态规划（DP）一轮只做引入，二轮才重兵投入？
**（答）** 作者本人当初在DP上硬刷到怀疑人生，每道题都靠题解，差点放弃算法。一轮只建立“DFS→记忆化→递推”的认知，保住你的信心。二轮再火力全开，那时候你已经有地图了。

### 4. 为什么反悔贪心/分层图篇幅较短？
**（答）** 分层图本质是“最短路+DP状态”，底子好了一天就能通；反悔贪心实战频率不高，且核心就一句“先选再踢”。学透思想即可，不必沉溺。节省的时间留给树形DP和区间DP，那才是压轴题主战场。
